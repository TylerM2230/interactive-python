1. What is a Tree?

A tree is a hierarchical data structure that consists of **nodes** connected by **edges**. Unlike linear structures (like lists, stacks, queues), trees represent relationships where items can have multiple successors (children). Think of a family tree or an organization chart.

Key Terminology:

  • Node: An element in the tree that holds data.
  • Edge: A connection between two nodes.
  • Root: The topmost node in the tree (the only node with no parent).
  • Parent: A node that has one or more nodes connected below it.
  • Child: A node connected directly below another node (its parent).
  • Leaf: A node with no children.
  • Subtree: A node and all its descendants form a subtree.
  • Height: The length of the longest path from the root to a leaf node.
  • Depth: The length of the path from the root to a specific node.

2. Why Use Trees?

Trees are used in many areas of computer science:

  • Representing Hierarchies: File systems, organization charts, family trees, XML/HTML document structure (DOM).
  • Efficient Searching: Binary Search Trees (BSTs) allow for fast searching, insertion, and deletion (often O(log n) on average).
  • Sorting: Tree-based sorting algorithms (like Heap Sort).
  • Decision Making: Decision trees in machine learning.
  • Database Indexing: B-trees and other tree variants are used to speed up database lookups.
  • Network Routing: Spanning trees in computer networks.

3. Binary Trees & Binary Search Trees (BST)

A **Binary Tree** is a specific type of tree where each node has at most two children, typically referred to as the "left child" and the "right child".

A **Binary Search Tree (BST)** is a special kind of binary tree with an ordering property:

  • For any given node:
  • All values in its **left subtree** are **less than** the node's value.
  • All values in its **right subtree** are **greater than** the node's value.
  • Both the left and right subtrees must also be binary search trees.

This ordering property allows for efficient searching, insertion, and deletion operations. 

# Conceptual structure of a BST node
class TreeNode:
  def __init__(self, key):
    self.key = key # The value stored in the node
    self.left = None # Reference to the left child node
    self.right = None # Reference to the right child node

5. Tree Traversal Methods

Traversal is the process of visiting (e.g., printing, processing) each node in a tree exactly once. Common depth-first traversal methods for binary trees are:

  • In-order Traversal: Visits the left subtree, then the current node, then the right subtree. For a BST, this visits nodes in ascending sorted order. (Left -> Root -> Right) 
    def inorder(node):
  if node:
    inorder(node.left)
    print(node.key) # Visit node
    inorder(node.right)
  • Pre-order Traversal: Visits the current node, then the left subtree, then the right subtree. Often used to create a copy of the tree or get prefix expressions. (Root -> Left -> Right) 
    def preorder(node):
  if node:
    print(node.key) # Visit node
    preorder(node.left)
    preorder(node.right)
  • Post-order Traversal: Visits the left subtree, then the right subtree, then the current node. Often used to delete nodes from a tree or get postfix expressions. (Left -> Right -> Root) 
    def postorder(node):
  if node:
    postorder(node.left)
    postorder(node.right)
    print(node.key) # Visit node
