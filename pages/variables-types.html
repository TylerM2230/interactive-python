Variables & Data Types

Learn how to store and manage different kinds of information in Python.

1. What are Variables?

In programming, a variable is like a labeled box where you can store a piece of information (data). You give the box a name (the variable name), and you can put different things in it, change what's inside, or just see what's inside. Think of it as a labeled storage box in a workshop. The label (e.g., player_score) tells you what kind of thing you expect to find inside, and the contents (e.g., 1000) are what's actually stored. Variables allow you to write more flexible and understandable code because you can refer to data by name. 

# Example:
character_name = "Pixel Knight"  # Storing text
character_level = 5              # Storing a whole number
health_points = 75.5           # Storing a number with a decimal
is_player_active = True          # Storing a True/False value

print(character_name)
print(character_level)

2. Common Data Types

Python can handle various types of data. Data types are like different categories of items you can store in your labeled boxes (variables). Python needs to know the type to understand how to work with the data correctly – you wouldn't try to perform a mathematical sum on a word in the same way you sum two numbers. Here are some of the most common ones:

  • Integers (int): Whole numbers, positive or negative, without decimals. They are like whole, indivisible items you can count – 3 apples, 10 gold coins, -5 penalty points. There are no fractions or pieces.
    Examples: -5, 0, 100, 9876 
    player_age = 25
  • Floating-Point Numbers (float): Numbers with a decimal point. These are for measurements that can have fractions or decimals, allowing for more precision than whole numbers – 75.5 health points, 19.99 gold pieces, 3.14159 (the value of pi).
    Examples: -3.14, 0.0, 2.718, 100.0 
    item_price = 19.99
  • Strings (str): Sequences of characters, used for text. Enclosed in single (') or double (") quotes. Strings are like messages written on a scroll or labels on items. They can be names ('Pixel Knight'), instructions ('Game Over!'), or even numbers you want to treat as text ('123 Main St'). The quotes are like the scroll itself, clearly marking where the text begins and ends.
    Examples: 'Hello', "Python Coder", "123 Main St" 
    message = "Game Over!"
  • Booleans (bool): Represent truth values, either True or False. Booleans are like simple light switches that can only be in one of two positions: ON (True) or OFF (False). They are crucial for making decisions in code – is the door open? (True), does the player have the key? (False).
    Examples: True, False 
    has_key = False
is_authenticated = True

Python is dynamically typed, meaning you don't have to explicitly declare the data type of a variable. Python figures it out automatically when you assign a value. 

# Python knows the type:
my_variable = 10         # my_variable is an int
my_variable = "Hello"    # Now, my_variable is a str (it changed!)
my_variable = 3.14       # Now, it's a float

# You can check a variable's type with the type() function:

3. Naming Variables (Rules and Conventions)

Choosing good variable names makes your code much easier to read and understand. It's like labeling your storage boxes clearly ('Old Photos,' 'Holiday Decorations') instead of using vague codes ('Box A,' 'Item 2'). Clear names help you (and others) instantly grasp what the variable holds.

Rules:

  • Must start with a letter (a-z, A-Z) or an underscore (_).
  • Can contain letters, numbers (0-9), and underscores.
  • Cannot be a Python keyword (e.g., if, for, while, class).
  • Are case-sensitive (myVariable is different from myvariable).

Conventions (Good Practices):

  • Use descriptive names (e.g., user_name instead of un or u_n).
  • For multi-word names, use "snake_case" (all lowercase with underscores), which is the common Python convention for variables and functions. This is like a preferred, consistent handwriting style for Python programmers – it makes all code look uniform and easier for everyone in the community to read. Examples: player_score, total_items, is_active.
  • Avoid using names of built-in functions or types if possible (e.g., don't name a variable list or print), as it can lead to confusion.
# Good names:
first_name = "Ada"
player_health = 100
is_game_over = False

# Bad names (avoid these):
# 1st_player = "Bob"  # Starts with a number
# player-score = 0    # Contains a hyphen (not allowed)
# for = "loop"        # 'for' is a keyword

4. Type Conversion (Casting)

Sometimes you need to convert data from one type to another. This is called type casting. Think of it like a workshop station where you can reshape or re-categorize an item. For example, you might have a number written down on a piece of paper (a string, like '150'), but you want to treat it as an actual quantity (an integer, 150) so you can perform mathematical calculations with it. Python provides built-in functions for this:

  • int(x): Converts x to an integer. This is like taking a price tag that says '$10' (text) and recognizing its numerical value as 10 (an integer).
  • float(x): Converts x to a float. This is like taking a measurement written as '7.5 inches' (text) and understanding it as the precise numerical value 7.5 (a float).
  • str(x): Converts x to a string. This is like taking a number, say your age 25 (an integer), and writing it down to include in a sentence: "My age is 25" (where "25" becomes part of the text).
score_text = "150"
score_number = int(score_text)  # Convert string "150" to integer 150
print(score_number + 10)        # Now you can do math: prints 160

level = 5
level_text = str(level)         # Convert integer 5 to string "5"
print("Current level: " + level_text) # Concatenate strings

# Be careful: you can't always convert.
# invalid_conversion = int("hello") # This would cause an error! You can't magically turn a word like 'apple' into a mathematical number.
